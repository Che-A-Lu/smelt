# Smelt Lite（炼知·轻）—— 起点

> 一个静态网页。拖文件进来变卡片。勾选卡片下载 `.card`。拖 `.card` 回来看到里面。三十秒。不需要安装、注册、AI 模型。

---

## 这是什么

**Smelt Lite** 是 Smelt 的第一个、最小的形态。它只做一件事：证明 `.card` 格式能跑通。

- 拖任何文件进来 → 生成卡片预览
- 勾选卡片 → 填个名字 → 下载 `.card`
- 拖 `.card` 进来 → 解包 → 看到里面有什么

就这些。没有画布。没有 AI。没有工作台。没有卡座。没有模式卡。

这是烧掉 10 亿 token 之后，从零开始重写的版本。Smelt Lite 大约 10 个源文件，几百行代码，一个明确的工作。

## .card 是什么

一张 `.card` 文件就是一个标准 ZIP 包。里面有：原始文件、完整的协作过程（每句 prompt、每次工具调用、AI 的思考链）、编辑记录、加密签名。改名 `.zip` 就能解压看到里面。

格式开放（MPL 2.0）。任何语言都能读写。不锁在任何平台。

## 为什么存在

做完一篇跟 AI 协作的论文之后，我发现除了 `.docx` 什么都没有——思考过程没了、推翻的路径没了、中间产物没了。我付出了时间和判断力，但带不走这个过程。

所有人都在做更好的 AI 工具。没有人在做「你做完之后带走的东西」。

**Smelt 不跟 AI 工具竞争。Smelt 收割你的对话——从任何 AI 工具——转成你拥有的 `.card` 文件。**

## 这不是什么

- 不是 AI 工作台
- 不是平台
- 不是 Chrome 扩展（还不是）
- 不是桌面应用（还不是）

## 下一步

Smelt 一步一步长，每个 `smelt-XX` 独立可用：

- **Smelt-Lite** ← 你在这里。静态网页，定义和验证 `.card`
- **Smelt-Ext** — Chrome 扩展。ChatGPT/Claude/Kimi 页面上一个按钮，一键导出 `.card`
- **Smelt-Desk** — 桌面应用。`.card` 成为原生文件类型，双击打开

每一步都独立可用。不等下一步。

## 快速开始

```bash
npm install
npm run dev        # → http://localhost:5173
npx tsc --noEmit   # 零错误
```
